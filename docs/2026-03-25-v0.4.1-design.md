# ZotPilot v0.4.1 Design Spec

**Created:** 2026-03-25
**Status:** Approved (Architect: APPROVE_WITH_NOTES, Critic: ACCEPT-WITH-RESERVATIONS)
**Branch:** dev → main via PR

---

## Summary

Patch release addressing v0.4.0 gaps: zero test coverage on bridge-dependent paths, OpenAlex as fallback instead of primary, and no user research context mechanism. Adds `profile_library` tool and rewrites the research workflow in SKILL.md.

---

## RALPLAN-DR

### Principles
1. **Backward compatibility** — no MCP tool signature changes; new fields are additive
2. **OpenAlex-first** — free, unlimited polite pool, returns OA metadata needed for smart ingest routing
3. **Test before ship** — bridge paths have zero coverage; fix before adding features
4. **Minimal footprint** — one new MCP tool (`profile_library`); scoring/ingest logic in SKILL.md
5. **Fail loud** — replace silent `except Exception: pass` with logged, typed errors

### Decision Drivers
1. **OA access** — `oa_url` and `landing_page_url` needed to choose optimal ingest path
2. **Reliability** — `_poll_one` swallows exceptions; S2-returns-0 never triggers OpenAlex
3. **Research context** — ZOTPILOT.md enables dynamic scoring without new server-side tools

### ADR: OpenAlex as Primary
- **Decision**: OpenAlex primary, S2 supplementary (only when `S2_API_KEY` set)
- **Why**: OpenAlex free/no-key, returns OA metadata, broader coverage; S2 rate-limits aggressively without key
- **Alternatives rejected**: Keep S2 primary (lacks OA metadata, returns 0 results for some queries); Remove S2 entirely (S2 has unique `s2_id`, citation graph value for users with API keys)
- **Consequences**: Additive fields (`is_oa`, `oa_url`, `landing_page_url`, `arxiv_id` extracted from OpenAlex); no breaking changes
- **Follow-ups**: OpenAlex ID as resolver input (backlog, post-v0.4.1)

## Phase 1: Test Coverage + Connector Robustness

**New file:** `tests/test_tools_ingestion_bridge.py`

### `_discover_saved_item_key` (6 tests)
- `known_key` provided → returns immediately
- No title and no URL → returns None
- Exactly 1 match → returns it
- 0 matches → returns None
- 2+ matches → returns None
- `find_items_by_url_and_title` raises → returns None (logged)

### `_apply_bridge_result_routing` (7 tests)
- No collection_key/tags → returns result unchanged
- item_key discovered, routing applied → collection + tags set
- item_key NOT discovered (0 matches) → returns warning
- Ambiguous match (2+) → returns warning
- No `ZOTERO_API_KEY` → returns warning about ignored routing
- `success=False` in result → no routing attempted
- item_key already in result (saveAsWebpage path) → skips discovery delay

### `save_from_url` (6 tests)
- Bridge not running → auto_start succeeds → enqueue → poll → result
- auto_start fails → returns error
- Enqueue 503 (extension_not_connected) → returns error with error_code
- Poll timeout (90s) → returns timeout error
- Enqueue network error → returns error
- Enqueue 200 but logical error in body → returns error

### `save_urls` (6 tests)
- Empty urls → ToolError
- >10 urls → ToolError
- auto_start fails → returns error
- 3 urls, all succeed → succeeded=3
- 3 urls, 1 enqueue fail + 1 timeout + 1 success → mixed results
- Enqueue 503 for first url → captured in enqueue_errors

**All bridge tests:** mock `BridgeServer.is_running`, `BridgeServer.auto_start`, `urllib.request.urlopen`; patch `time.sleep`

### Fix: silent exception swallowing
- `save_from_url` poll loop (line 467): `except Exception: pass` → `logger.debug(f"Poll {request_id}: {e}")`
- `save_urls` `_poll_one()` (line 558): same fix
- 204 responses: handled as normal polling continuation, no warning logged
- Distinct, actionable error message per failure mode

## Phase 2: `search_academic_databases` Refactor

**File:** `src/zotpilot/tools/ingestion.py`

### Control flow (new)
```
Always call _search_openalex() first
If S2_API_KEY set:
  call S2, merge by DOI (lowercase-normalized)
  if S2 fails: log, continue with OpenAlex-only
If OpenAlex fails:
  try S2 as fallback (if key set)
  if both fail: raise ToolError
```

### `_search_openalex` additions
Add to `select` param: `primary_location`, `ids`, `open_access`

New fields per result:
- `is_oa`: `p.get("open_access", {}).get("is_oa", False)`
- `oa_url`: `p.get("open_access", {}).get("oa_url")`
- `landing_page_url`: `p.get("primary_location", {}).get("landing_page_url")`
- `arxiv_id`: extracted from `p.get("ids", {})` — check `arxiv` key (verify exact field path against OpenAlex API during implementation)

### Merge semantics (OpenAlex + S2)
- Deduplicate by `doi.lower()` — OpenAlex result is base, S2 enriches with `s2_id` and `citationCount` where missing
- Papers without DOI: kept as separate entries (no fuzzy merge in v0.4.1)
- S2-only fields default to `None`/`False` on OpenAlex-only results

### Config wiring
- `openalex_email`: `config.openalex_email` → mailto param; fallback to `"zotpilot@example.com"` (not empty string — preserves polite pool)

### `ingest_papers` updates
- Remove/update S2 rate-limit warning (lines 326-331) — no longer accurate after OpenAlex-primary
- Identifier priority unchanged: `doi > arxiv_id > s2_id` — `arxiv_id` now populated from OpenAlex, so coverage improves without resolver changes

### New tests (Task 2.3)
- OpenAlex succeeds alone (no S2 key)
- OpenAlex + S2 merge: same DOI → single deduplicated result with both `s2_id` and `oa_url`
- OpenAlex + S2 merge: DOI case mismatch → still deduplicates
- OpenAlex fails, S2 fallback succeeds
- Both fail → ToolError with both error messages
- OA fields present in OpenAlex results
- `openalex_email` config value used as mailto

## Phase 3: `profile_library` Tool

**File:** `src/zotpilot/tools/library.py`

### Tool contract
```python
@mcp.tool()
def profile_library() -> dict:
    """Analyze the Zotero library to generate a user profile for research context.
    Returns library statistics for use in the profiling workflow (SKILL.md).
    Pure read operation — no side effects."""
```

### Return schema
```json
{
  "total_items": 312,
  "year_distribution": {"2024": 45, "2023": 67, ...},
  "top_tags": ["machine learning", "NLP", ...],
  "top_collections": [{"name": "...", "key": "...", "count": ...}],
  "topic_density": {"indexed": true, "doc_count": 312, "section_types": {...}},
  "gaps": ["sparse coverage before 2015", "few survey papers"],
  "existing_profile": "<contents of ~/.config/zotpilot/ZOTPILOT.md if exists, else null>"
}
```

- `existing_profile`: reads `~/.config/zotpilot/ZOTPILOT.md` if it exists; returns contents as string (not null). Avoids agent filesystem dependency across MCP clients.
- `topic_density`: from `_get_store_optional()` if index available; `{"indexed": false}` in no-RAG mode
- `gaps`: agent-interpretable strings derived from year_distribution and tag sparsity

### Tests (3+)
- Library with items → year distribution, tags, collections populated
- No-RAG mode (store_optional returns None) → metadata fields still returned, topic_density.indexed=false
- Empty library → zeros/empty lists
- Existing ZOTPILOT.md → returned in `existing_profile`

## Phase 4: SKILL.md Updates

### New section: Library Profiling

```
**Library profiling & ZOTPILOT.md:**
1. call profile_library() → get library analysis + existing_profile
2. Show draft analysis to user: "这是我对你文献库的分析，准确吗？"
3. User confirms/corrects
4. Interview (one question at a time):
   a. 学科领域？
   b. 身份？(PhD / Master / Researcher / Other)
   c. 主要研究方向？(1-3个)
   d. 关注的交叉学科？
5. Write ~/.config/zotpilot/ZOTPILOT.md
```

**ZOTPILOT.md structure:**
```markdown
# ZotPilot User Profile
generated: YYYY-MM-DD

## Identity
- Role: PhD
- Discipline: Computer Science
- Cross-disciplinary interests: Cognitive Science

## Research Focus
- Primary: Multimodal LLMs, Vision-Language Models
- Secondary: Efficient inference

## Library Analysis
- Total indexed: 312 papers
- Year distribution: 70% 2021-2025, 20% 2016-2020, 10% pre-2016
- Topic density: LLM (high), CV (medium), Bio (sparse)
- Gaps: few survey papers, weak on pre-transformer foundations
```

### Dynamic scoring (replaces Step 2 in research workflow)

**Default weights** (no special intent signal):

| Dimension | Weight |
|-----------|--------|
| Query relevance | 40% |
| User context fit (ZOTPILOT.md) | 30% |
| Quality signal (citations, source type) | 20% |
| Recency | 10% |

**When no ZOTPILOT.md exists:** context fit → 0%, relevance → 70%

**Intent-driven weight adjustments:**

| Intent signal | Relevance | Context | Quality | Recency |
|---------------|-----------|---------|---------|---------|
| "最新" / "recent" | 35% | 25% | 10% | 30% |
| "经典" / "foundational" | 35% | 25% | 30% | 10% |
| "探索新方向" / "exploring" | 60% | 0% | 25% | 15% |
| "高引" / "high impact" | 35% | 25% | 35% | 5% |
| "综述" / "survey" | 50% | 30% | 15% | 5% + type filter |

Intent recognition: agent judges from natural language, not keyword matching.

**Display format:**
```
1. [9.2] Attention Is All You Need (Vaswani et al., 2017) · 8420引用 · OA
   "直接奠定你研究的 transformer 基础，与你的 VLM 方向高度契合"
```

### 5-tier ingest priority (replaces current decision table)

```
1. arxiv_id present          → add_paper_by_identifier("arxiv:ID")        OA PDF, fast
2. doi + is_oa + oa_url      → add_paper_by_identifier(doi, oa_url)       OA PDF direct
3. doi + no OA + landing_url → save_urls([landing_page_url])               Connector, ~30s
4. doi only (no URL)         → ingest_papers([{doi:...}])                  metadata only
5. no doi                    → skip, inform user                           (openalex_id: backlog)
```
